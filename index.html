    CH01 — Market Structure (BOS / CHoCH)

    1) Objective

    ยืนยันทิศหลัก (bias) ด้วยหลักฐานของแรง: BOS/CHoCH

    2) Process

    1. กำหนด swing HL/HH (ขึ้น) หรือ LH/LL (ลง) ล่าสุดให้ชัด
    2. รอแท่งปิด “ทำลาย” โครงสร้าง (BOS) → ห้ามเดา
    3. ใช้ CHoCH เป็นสัญญาณเตือน “เปลี่ยมมือ”

    3) What to Watch

    • แท่ง displacement ชัด
    • Follow-through หลัง BOS

    4) Why

    BOS แปลว่าฝ่ายหนึ่งชนะจริง (order flow shift)

    5) Branch

    • ถ้า BOS ไม่ชัด → ไม่ตั้ง bias
    • ถ้า CHoCH แต่ไม่มี follow → ถือเป็นสัญญาณเตือน

    6) Mistakes

    • รีบรอ BOS
    • ตีโครงสร้างกลาง range

    7) Worked Example

    ตัวอย่าง BOS (เส้นเขียว) + micro BOS ก่อน Entry

    Deep Why: เรา “ซื้อหลักฐาน” ไม่ใช่ “ซื้อลางสังหรณ์” — BOS คือหลักฐาน

    SNF Quick: S=8 N=7 F=7 T=8 → รวม 30/40 (เข้าได้)

    Checklist: BOS ปิดแท่งแล้ว? อยู่ฝั่ง value? มี micro BOS? เวลาใช่?

    8) Summary

    • รอ BOS จริง
    • CHoCH = เตือน
    • โยงไปหา value zone ต่อ

    9) Reflection & Homework

    เปิดกราฟย้อนหลัง 3 วัน วง BOS/CHoCH ทั้งหมด พร้อมเหตุผล

    CH02 — Liquidity Map (PDH/PDL • Asia HL • EQH/EQL)

    Objective

    วาง “แผนที่ราคา” ก่อนมองจุดเข้า

    Process

    1. ตี PDH/PDL
    2. กรอบ Asia
    3. หา EQH/EQL
    4. โซนคาดการณ์ sweep

    What to Watch

    • sweep → BOS สวน
    • FVG/OB ฝั่งใหม่

    Why

    ราคาไปหา “ของเหลือ” เสมอ (resting liquidity)

    Mistakes

    • เข้า “ระหว่างกวาด”

    PDH/PDL • Asia • EQL + จุดที่มัก sweep ก่อนกลับ

    CH03 — Fair Value Gap (Displacement & 50%)

    Objective

    ใช้ช่องว่างที่เกิดจากแรง “ไม่สมดุล” เป็นจุดรับ/ยิง

    Process

    1. หาแท่ง displacement
    2. วาดกล่อง FVG + เส้น 50%
    3. รอ micro BOS ในกล่อง

    Why

    ตลาดมีแนวโน้มกลับมา “ทักทาย” กลาง FVG

    FVG50% + micro BOS

    CH04 — Premium / Discount (Dealing Range)

    Objective

    ซื้อฝั่ง Discount ในขาขึ้น / ขายฝั่ง Premium ในขาลง

    Process

    1. ลาก Fib 0–100 ตามทิศ
    2. เลือกฝั่ง value ที่ตรง bias

    Why

    ทำให้ RR ดีขึ้นและเข้าฝั่ง “มีอัตราได้เปรียบ”

    CH05 — Order Block (Fresh vs Mitigated)

    Objective

    ใช้ OB สดเป็นโซนตั้งรับ/ยิงในทิศที่ยืนยันแล้ว

    Process

    1. หา OB ก่อน displacement
    2. คัด “สด” (ไม่ถูกแตะ)
    3. รอ tap + micro BOS

    Mistakes

    • OB ใหญ่เกิน
    • OB ที่ขัด value

    CH06 — Timing (Asia • London • New York • News)

    Objective

    กรองคุณภาพสัญญาณด้วยเวลา

    Process

    1. ระบุ Killzone
    2. จดข่าวแรง
    3. ตั้งหมดอายุสัญญาณ ≤ 3 แท่งถ้าไม่แตะโซน

    Asia=สะสม • London=กวาด • NY=ต่อเนื่อง/กลับตัว

    PART B — Trade Setup Bible (จับมือทำทีละขั้น)

    ทุก Setup เดินตามกรอบเดียวกัน: Preconditions → Identify → Stalk → Confirm → Execute → Manage → Reflect + Entry/SL/TP และ SNF Score

    SETUP #1 — Sweep → BOS → FVG (Reversal)

    Preconditions

    • HTF bias ชัด
    • มี Liquidity ใกล้ (PDH/PDL/EQH/EQL)

    Identify

    รอ sweep โซน → มองหา displacement สวน

    Stalk

    หา FVG50% ในฝั่งใหม่ + OB สด

    Confirm

    micro BOS (5m/1m) + เวลาเป็นใจ (Killzone)

    Execute

    • Entry: Tap FVG50% หรือ OB สด
    • SL: หลัง OB / นอก sweep wick
    • TP: TP1 ที่ resting liquidity ถัดไป • TP2 opposing OB

    Manage

    • BE ที่ +1R
    • Partial 30–50%

    Reflect

    วิเคราะห์สาเหตุที่ win/loss → ปรับกติกา

    SETUP #2 — OB-in-FVG (Continuation)

    Identify

    FVG ในเทรนด์หลัก + OB สดซ้อนภายใน

    Confirm

    Tap + micro BOS

    Execute

    เข้า OB สด, SL หลัง OB, TP ตาม value ถัดไป

    Manage

    ย้าย BE หลัง +1R

    SETUP #3 — Session Sweep + Inducement

    Idea

    London/NY มักกวาดกรอบ Asia ก่อนเลือกทิศ

    Flow

    1. sweep Asia HL
    2. BOS สวน
    3. FVG50% + micro BOS
